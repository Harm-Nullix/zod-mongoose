# Overview

`@nullix/zod-mongoose` is a monorepo containing the `@nullix/zod-mongoose` ecosystem, a collection of tools for integrating [Zod](https://github.com/colinhacks/zod) with [Mongoose](https://github.com/Automattic/mongoose).

## Why mongoose-zod?

Declaring Mongoose schemas in a TypeScript environment has always been challenging in terms of type safety:
* You either have to first declare an interface representing a document in MongoDB and then create a schema matching that interface (resulting in no type safety at all).
* Or you reverse the process by using `mongoose.InferSchemaType<typeof schema>`, which is far from ideal.
* Finally, you could use [typegoose](https://github.com/typegoose/typegoose), which is based on the outdated decorators proposal and generally has poor type inference.

This library aims to solve many of the above problems by using `zod` as the tool for schema composition.

## Key Features

- **Zod v4 Support**: Optimized for the latest Zod features and registry.
- **Native Discriminators**: Automatically maps `z.discriminatedUnion` to native Mongoose discriminators.
- **XOR Support**: Implements mutual exclusivity validation for non-inclusive unions.
- **Isomorphic**: Use the same schemas on the frontend and backend with `setFrontendMode`.
- **Nuxt 4 Ready**: Built-in support for Nuxt and Nitro environments.
- **Automatic Validation Mapping**: Automatically maps Zod's built-in validations to their corresponding Mongoose SchemaType options.
